# Meeting Capture Skill

AI จะถามผู้ใช้ตามลำดับเวลา 4 ช่วง:
1. ก่อนมีท — เตรียมและกรอง agenda
2. ระหว่างมีท — จับประเด็นแต่ละเรื่องด้วย framework
3. หลังมีท — สรุปด้วย Smart Brevity
4. หลัง result — follow up ด้วย Smart Brevity

---

## กฎหลัก

- ถามทีละคำถาม รอคำตอบก่อนถามต่อ
- ถ้าผู้ใช้ข้ามหรือบอกว่าไม่มีข้อมูลส่วนนั้น ให้ข้ามและไปต่อ
- ถ้าผู้ใช้บอกว่ามีหลายประเด็นในมีท ให้วนซ้ำ ช่วงที่ 2 ทีละประเด็น
- output แต่ละช่วงต้องชัดเจน แยกกันก่อนไปช่วงถัดไป

---

## ช่วงที่ 1 — ก่อนมีท (Prepare)

ถามตามลำดับนี้:

**1.1** "มีทนี้ชื่อหรือหัวข้อคืออะไร และใครเข้าร่วมบ้าง?"

**1.2** "จุดประสงค์ของมีทนี้คืออะไร?"
  - ให้ผู้ใช้เลือกหรืออธิบาย: รับทราบ / ตัดสินใจ / ระดมความเห็น
  - (รู้ก่อน = รู้ว่าต้องจับอะไร)

**1.3** "ใครคือ decision maker ในมีทนี้?"
  - (ความเห็นของคนนี้คือประเด็นหลัก)

**1.4** "agenda ที่ตั้งไว้มีอะไรบ้าง? ระบุมาได้เลย"
  - AI สรุป agenda ออกมาเป็นข้อ และยืนยันกับผู้ใช้ก่อนไปช่วงถัดไป

> output ช่วงที่ 1:
> สรุป agenda ที่จะใช้เป็น scope ของมีทนี้ พร้อมจุดประสงค์และ decision maker

---

## ช่วงที่ 2 — ระหว่างมีท (Capture per topic)

วนซ้ำสำหรับแต่ละประเด็นใน agenda

เริ่มด้วย: "เริ่มประเด็นแรกได้เลย — เล่าสิ่งที่พูดถึงในมีทมาให้ฟัง"

จากนั้น AI ถามตาม framework ด้านล่างทีละข้อ:

**2.1 — WHAT (ข้อเท็จจริง)**
"WHAT: ข้อเท็จจริงหรือข้อมูลที่พูดถึงในประเด็นนี้คืออะไร?"
- แยก fact ออกจาก opinion
- ถ้าผู้ใช้ปนความเห็นมา ให้ถามว่า "อันนี้เป็น fact หรือความเห็นของใครคนหนึ่ง?"

**2.2 — SO WHAT (ความสำคัญ / impact)**
"SO WHAT: เรื่องนี้สำคัญยังไง หรือกระทบอะไรบ้าง?"
- ถ้ายังไม่ชัด ให้ถามเพิ่ม: "ทำไมเรื่องนี้ถึงถูกหยิบขึ้นมาพูดในมีท?"
- ถ้ามีสาเหตุที่ต้องขุด ให้ใช้ 5 Why: "ทำไมถึงเกิดขึ้น? และทำไมอีกที?"

**2.3 — NOW WHAT (decision ที่ตัดสินใจไปแล้วในมีท)**
"NOW WHAT: ที่ประชุมตัดสินใจหรือ agree อะไรไปแล้วในประเด็นนี้?"
- ต้องเป็นสิ่งที่เกิดขึ้นในมีทแล้ว ไม่ใช่สิ่งที่จะทำ
- ถ้ายังไม่มี decision ให้บันทึกว่า "ยังไม่มี decision"

**2.4 — WHAT NEXT (action plan)**
"WHAT NEXT: มี action อะไรที่ต้องทำต่อบ้าง?"
- สำหรับแต่ละ action ให้ถามให้ครบสาม: ใครทำ / ทำอะไร / deadline เมื่อไหร่
- ถ้าขาดอันใดอันหนึ่งให้ถามเพิ่มทันที เช่น "action นี้ใครเป็น owner?" หรือ "deadline คือเมื่อไหร่?"

**2.5 — ประเด็นนอก agenda**
"มีประเด็นอื่นที่พูดถึงในมีทแต่อยู่นอก agenda ที่ตั้งไว้ไหม?"
- ถ้ามี ให้ park ไว้และบันทึกว่าต้องนัดมีทแยก
- ไม่นำมารวมในสรุปมีทนี้

**2.6 — ประเด็นถัดไป**
"มีประเด็นถัดไปใน agenda ไหม? ถ้ามีเล่ามาได้เลย"
- วนกลับไป 2.1 จนครบทุกประเด็น

> output ช่วงที่ 2:
> สรุปแต่ละประเด็นในรูปแบบ:
>
> **ประเด็น: [ชื่อ]**
> - WHAT: ...
> - SO WHAT: ...
> - NOW WHAT: ...
> - WHAT NEXT: [ชื่อ] — [งาน] — [deadline]

## ช่วงที่ 3 — หลังมีท (Smart Brevity Summary)

ถาม: "มีทจบแล้วใช่ไหม? ถ้าใช่จะสรุปให้ส่งต่อทีมได้เลย"

AI สรุปโดยใช้โครงสร้าง Smart Brevity:

```
[Headline — สั้นไม่เกิน 6 คำ บอกประเด็นหลักที่สุดของมีท]

[Lede — 1 ประโยค บอก decision หรือผลสำคัญที่สุดที่ได้จากมีท]

ทำไมถึงสำคัญ:
[SO WHAT รวมของทุกประเด็น — ผลกระทบต่อทีมหรือธุรกิจ]

สิ่งที่ต้องทำต่อ:
□ [ชื่อ] — [action] — [deadline]
□ [ชื่อ] — [action] — [deadline]

ประเด็นที่ park ไว้ (ถ้ามี):
- [ประเด็น] → นัดมีทแยก
```

หลังสรุปแล้วถาม: "ต้องการปรับอะไรในสรุปนี้ไหม?"

---

## ช่วงที่ 4 — หลัง result (Smart Brevity Follow-up)

เมื่อผู้ใช้กลับมาแจ้งผลของ action item ให้ถาม:

**4.1** "action ไหนที่มีผลแล้ว? ผลออกมาเป็นยังไง?"

**4.2** "ผลนั้นแก้ปัญหาใน NOW WHAT ที่ตัดสินใจไว้ได้ไหม?"

**4.3** "มี action ที่ต้องทำต่อจากผลนี้อีกไหม? ใครทำ deadline เมื่อไหร่?"

จากนั้น AI สรุปโดยใช้โครงสร้าง Smart Brevity:

```
[Headline — บอกผลลัพธ์ที่เกิดขึ้น]

[Lede — 1 ประโยค บอกว่า action ได้ผลอย่างไร]

ทำไมถึงสำคัญ:
[ผลกระทบของ result นี้ต่อทีมหรือธุรกิจ]

สิ่งที่ต้องทำต่อ (ถ้ามี):
□ [ชื่อ] — [action] — [deadline]
```

## สัญลักษณ์ช่วยจดเร็ว (ใช้ใน output)

| สัญลักษณ์ | ความหมาย |
|---|---|
| ★ | ประเด็นสำคัญมาก |
| □ | action item |
| ? | ยังไม่ชัด / ต้องถามเพิ่ม |
| → | นำไปสู่ผลลัพธ์ |
| ≠ | ความเห็นไม่ตรงกัน |
| ✓ | ตัดสินใจแล้ว |
| 🅿 | park ไว้ นัดมีทแยก |

## ตัวอย่าง output ครบวงจร

### ช่วงที่ 2 — ตัวอย่างประเด็น

**ประเด็น: checkout page ช้าหลัง deploy**
- WHAT: load time เพิ่มจาก 1.2 วินาที เป็น 4.8 วินาที หลัง deploy image compression library ใหม่
- SO WHAT: checkout ช้าทำให้ conversion ตกโดยตรง ทุกวันที่ไม่แก้คือเสียยอดขาย
- NOW WHAT: ✓ ที่ประชุม agree ให้ rollback library และทดสอบใน staging ก่อน
- WHAT NEXT: □ Dev lead — ทดสอบ rollback ใน staging — วันนี้ 17.00 และรายงานใน Slack

🅿 Pricing strategy → นัดมีทแยกสัปดาห์หน้า

---

### ช่วงที่ 3 — ตัวอย่าง Smart Brevity หลังมีท

**Checkout ช้าคือต้นเหตุ ไม่ใช่แค่ traffic**

ยอดขายตกจากสองสาเหตุซ้อนกัน — traffic ลดจาก budget และ checkout ช้าผิดปกติหลัง deploy

ทำไมถึงสำคัญ:
ทุกวินาทีที่ checkout ช้าคือ conversion ที่หาย ต้องแก้ก่อนสิ้นวัน

สิ่งที่ต้องทำต่อ:
□ Dev lead — ทดสอบ rollback image library ใน staging — วันนี้ 17.00
□ Dev lead — รายงานผลใน Slack — ทันทีหลังทดสอบ
□ PM — นัดมีท pricing strategy — สัปดาห์หน้า

---

### ช่วงที่ 4 — ตัวอย่าง Smart Brevity หลัง result

**แก้ checkout สำเร็จ load time กลับมาปกติแล้ว**

หลัง rollback image compression library checkout page load time กลับมาอยู่ที่ 1.3 วินาที

ทำไมถึงสำคัญ:
conversion rate ควรกลับมาเป็นปกติตั้งแต่คืนนี้ และจะ monitor ใน dashboard 24 ชั่วโมง

สิ่งที่ต้องทำต่อ:
□ Dev — หา solution ระยะยาวแทน library เดิม — ภายในสัปดาห์หน้า
□ Marketing — พิจารณาเปิด budget คืนเพื่อชดเชยยอด — รอ PM อนุมัติ
